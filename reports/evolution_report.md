## Interpretation

- This report treats the KG as an evolving artifact rather than a one-off export.
- Changes in links, labels, and inferred facts are visible between snapshots.
- A non-zero `rule_fact_delta` is the LO8 evidence that the rule layer reacts to a change in
  the rule definitions or the underlying data. In the snapshot shipped with this repo the delta
  comes from lowering the `professional_operator` threshold from 4 to 3 in `src/rules.yml`,
  which is exactly the kind of curatorial tuning the snapshot mechanism is designed to track.
- To produce a fresh non-zero delta, change a threshold in `src/rules.yml` (or add a new
  Wikidata row), then re-run `python src/materialize_rules.py && python src/version_snapshot.py && python src/diff_snapshots.py`.
